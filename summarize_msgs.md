# summarize_msgs.py — .msg 폴더 요약 리포트 생성기

`mail_search_to_msg.py` 로 받은 `.msg` 파일들을 한 개의 마크다운 리포트로 정리하는 휴리스틱 기반 요약 도구.

기준 문서 업데이트일: 2026-05-11

## 무엇을 하는가

지정된 폴더의 모든 `.msg` 를 시간순으로 정렬해서, 메일별로 다음을 추출:

- **메타데이터** — 보낸 시각 / From / To / Cc / 첨부 / 원본 파일명
- **🔑 액션 아이템 후보** — 본문에서 `요청 / 부탁 / 이슈 / TODO / 확인 / 마감 / ...` 같은 키워드를 포함한 줄
- **📝 본문 미리보기** — forwarded thread 시작 지점까지만 (인용 `>`줄·연속 공백 제거)

상단에는 메일 전체 통계 + 통합 섹션:

- 메일 수, 기간 (최초~마지막)
- 발신자 TOP N (건수 내림차순)
- 액션 키워드 빈도 (전체 본문 합산)
- **🔑 액션 아이템 통합 — 키워드별** : 모든 메일의 키워드 매칭 줄을 한 곳에 모아 키워드별로 묶음. 각 줄에 `#NN` 메일 번호가 붙어 있어 타임라인 섹션으로 점프 가능

## 의존성

## 사용

스크립트 상단의 설정 상수만 본인 환경에 맞게 두고 실행:

```bash
python summarize_msgs.py
```

| 상수 | 기본값 | 의미 |
|---|---|---|
| `SOURCE_DIR` | `~/Downloads/mail_search_YYMMDD` | 파싱할 `.msg` 폴더 (보통 `mail_search_to_msg.py` 출력 위치) |
| `OUTPUT_DIR` | `None` | `None` 이면 SOURCE_DIR 안에 저장 (`.msg` 와 같이 묶임). 별도 경로 박으면 거기로 |
| `OUTPUT_BASENAME` | `_summary` | 출력 파일명 base. 정렬용으로 `_` 가 prefix — `_summary_YYMMDD_HHMM.md` 형태로 저장 |
| `ACTION_KEYWORDS` | 한/영 혼합 다수 | 이 키워드가 들어있는 줄을 액션 아이템 후보로 추출 |
| `BODY_PREVIEW_LINES` | 25 | 본문 미리보기 최대 줄 수 |
| `MAX_LINE_LENGTH` | 200 | 너무 긴 한 줄은 잘라서 표시 |
| `TOP_SENDERS` | 10 | 통계 섹션 발신자 TOP 개수 |
| `INCLUDE_FULL_BODY` | `False` | True 면 메일별 `<details>` 블록에 본문 전체 포함 (디버그) |

## 동작 방식 — 본문 정제

각 메일 본문은 아래 순서로 손질됨:

1. 줄 단위 split (`\r\n`/`\r`/`\n` 통일)
2. **Forwarded thread 절단점** 만나면 그 이후 버림
   - `보낸 사람:`, `From:`, `발신:`, `--- Original Message ---`, `____...`, `====...`, `------...` 등
3. **인용 줄(`>`)** 제거
4. **연속 빈 줄** 1개로 압축
5. **너무 긴 줄**은 `MAX_LINE_LENGTH` 에서 잘라 `…` 부착

→ 결과적으로 "이 메일에서 새로 작성한 본문" 만 남는다 (덧붙은 회신 체인은 안 보임).

## 결과 파일 예

```
_summary_YYMMDD_HHMM.md     # 예: _summary_260105_1735.md
```

## 매 실행마다 파일이 어떻게 교체되는가 — M365 링크 보존

같은 폴더에 `_summary_*.md` 가 누적되지 않고 **항상 1개만** 남도록 다음 로직:

1. 실행 시점 timestamp 로 새 이름 결정 (`_summary_YYMMDD_HHMM.md`)
2. 폴더 안의 기존 `_summary_*.md` 가 있으면:
   - 여러 개 있으면 가장 최근 1개만 keeper 로 살리고 나머지 삭제
   - keeper 가 새 이름과 다르면 **rename (file ID 유지)** → OneDrive/M365 sharing 링크 살아있음
3. 그 파일에 새 내용 덮어쓰기

→ 결과: 파일명에 최근 실행 시각이 박혀있고, 공유 링크는 안 깨짐. 단, 드물게 OneDrive 동기화 엔진이 rename 을 delete+create 로 처리하는 케이스가 있을 수 있음 (락 걸려있을 때 등).

## 키워드 추가/수정

`ACTION_KEYWORDS` 리스트에 줄 단위로 추가하면 됨. 매칭은 **부분 문자열, 대소문자 무관**:

```python
ACTION_KEYWORDS = [
    "요청", "부탁", ...
    # 새로 추가
    "리뷰", "approval", "blocker",
]
```

> 캠페인/프로젝트 약어를 본문에서 자주 보면 키워드로 추가해도 좋지만, 너무 흔한 단어는 노이즈가 늘어남.

## 한계

- **요약이 아니라 추출** — 휴리스틱 기반이라 "이 스레드의 결론은 X" 같은 진짜 요약은 못 함. 키워드 줄들을 빠르게 훑는 용도.
- **Forward marker 못 잡는 케이스** — 회신 본문 형식이 특이하면 전체 본문이 미리보기로 들어옴. `FORWARD_MARKERS` 패턴에 정규식 추가하면 됨.
- **본문 외 정보 부족** — 첨부파일 내용은 안 봄. 일정 엑셀 안의 셀 값까지 보려면 별도 처리 필요.
- **요청/지시 톤 구별 안 함** — "확인 부탁드립니다" 도 "이미 확인했습니다" 도 둘 다 `확인` 키워드로 잡힘. 진짜 액션이 필요한 줄만 보고 싶으면 결과 MD 에서 한 번 더 눈으로 필터.

## 향후 확장 아이디어

- **첨부 인덱싱** — 같은 폴더의 첨부 파일을 메일별로 묶어서 표시 (현재는 메일 자체에 포함된 attachments 만 표시)
- **스레드 그룹핑** — `RE:` / `Fw:` 제거한 정규화 제목으로 그룹화 후 스레드별 timeline
- **LLM 요약** — 본문을 Claude API 로 보내 "이 메일의 핵심 요청 1줄" 추출. 30개 × 토큰 비용 발생.
